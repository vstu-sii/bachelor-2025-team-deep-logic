# Итоги по требованиям — AI Personal Chef

## 1. Введение
Цель документа — зафиксировать текущее состояние требований проекта после реализации прототипа (ЛР‑3), проверить соответствие утверждённым Use‑cases (UC1–UC9) и отметить отклонения.

---

## 2. Проверка соответствия Use‑cases (из ЛР‑2 → ЛР‑3)

| Use‑case | Статус выполнения | Комментарий |
|----------|------------------|-------------|
| UC1: Регистрация | ✅ Реализовано | UI и Backend готовы, база (SQLite) подключена, авторизация после регистрации работает |
| UC2: Вход в систему | ✅ Реализовано | Форма входа, проверка данных и доступ к функционалу реализованы |
| UC3: Просмотр истории | Частично ✅ | SQLite подключен, UI есть, end‑to‑end сценарий не завершён |
| UC4: Управление профилем | Частично ✅ | UI реализован, учёт предпочтений при подборе рецептов не завершён |
| UC5: Запрос списка рецептов по ингредиентам | Частично ✅ | Фото загружается, VLM протестирован, интеграция в пайплайн не завершена |
| UC6: Исключение ингредиентов | ✅ Реализовано | UI и Backend поддерживают исключение, генерация списка рецептов работает |
| UC7: Выбор рецепта | Частично ✅ | Список рецептов отображается, план приготовления тестировался, повторная генерация не завершена |
| UC8: Оставление отзыва | ❌ Не реализовано | Отложено, признано техническим долгом |
| UC9: Добавление в избранное | Частично ✅ | UI реализован, сохранение в SQLite есть, обработка ошибок не завершена |

---

## 3. Валидация use‑cases через прототип
- **Что проверено:**  
  - UC1, UC2, UC3, UC5, UC6, UC7, UC9 — проверены на уровне UI и отдельных вызовов.  
  - Отдельные вызовы VLM (Ollama) и LLM (Mistral API) работают.  
  - SQLite подключен, базовые операции сохранения возможны.  
- **Что не проверено:**  
  - Полный end‑to‑end сценарий (фото → ингредиенты → рецепт → история → избранное).  
  - UC8 (отзывы).  
  - Уведомления и UI‑сценарии.  

---

## 4. Анализ отклонений от плана
- Реализация фронтенда отстаёт от плана (UI отсутствует для части сценариев).  
- Интеграция компонентов (Backend ↔ RabbitMQ ↔ Model Manager ↔ VLM/LLM) не завершена.  
- UC8 (Отзывы) перенесён в технический долг.  
- UC9 реализован частично (UI и база), но требует доработки обработки ошибок.  
- MVP‑сценарий пока не работает сквозным образом.  

---

## 5. Обновление документации требований
- Требования UC1–UC9 остаются актуальными, но их статус уточнён (реализовано/частично/не реализовано).  
- UC8 (Отзывы и оценки) перенесён в технический долг.  
- UC9 (Избранное) включён в план доработки (ошибки при добавлении в избранное).  
- Добавлено ограничение: уведомления реализуются только для Android (FCM), iOS (APNS) исключены.  
- В документации требований необходимо обновить статусы и отметить отклонения.  

---

## 6. Выводы
- Прототип частично подтверждает требования из ЛР‑2.  
- Основные функции (регистрация, вход, исключение ингредиентов, распознавание, генерация рецептов, сохранение истории, избранное) протестированы, но не связаны в end‑to‑end сценарий.  
- UC8 остаётся нереализованным.  
- Для достижения MVP в Спринте 4 необходимо завершить интеграцию компонентов и реализовать базовый пользовательский сценарий (фото → ингредиенты → рецепт → история → избранное).
