## 2. Quality Gates

Quality Gates — это проверки качества, которые должны проходить все изменения в проекте.

### Сейчас
- ❌ Автоматических тестов пока нет  
- ❌ CI/CD пайплайн не настроен  
- ✅ Код проверяется вручную (стиль, ошибки)  

### План
- Добавить простые unit‑тесты для основных функций (регистрация, вход, рецепты)  
- Сделать интеграционные тесты для сквозного сценария (от фото до избранного)  
- Настроить сбор покрытия тестами (например, минимум 70%)  
- Настроить CI/CD (GitHub Actions), чтобы проверки запускались автоматически  
- Добавить проверку зависимостей и секретов  

**Вывод:** пока качество контролируется вручную, но есть план внедрить автоматические проверки.

- Время распознавания фото ≤ 1,5 минуты.    
- Нагрузка: система выдерживает 3 параллельных запросов без деградации

## 4. Security Audit (базовый)

### Текущее состояние
- ❌ Пароли хранятся в базе в открытом виде (plain text), проверка идёт простым сравнением строк   
- ❌ Нет мониторинга попыток взлома  
- ❌ Нет ограничений на количество попыток входа  
- ✅ Конфигурация и секреты вынесены в `.env` файл  

### Риски
- Высокая вероятность утечки данных при компрометации базы  
- Возможность подбора паролей (brute force) без ограничений  
- Отсутствие базовых практик безопасности (хэширование, salting, токены доступа)

**Вывод:** безопасность на текущем этапе минимальная. Для прототипа это допустимо, но переход к продакшену невозможен без внедрения хэширования паролей и базовых мер защиты.
